You are an ML researcher investigating whether transcoders — a recent alternative to sparse autoencoders — can decompose vision transformer representations into interpretable features.

## Background

Sparse autoencoders (SAEs) decompose neural network activations into sparse, interpretable features by learning to reconstruct an activation from itself. **Transcoders** are a variant proposed in "Transcoders Beat Sparse Autoencoders for Interpretability" (Paulo et al., 2025) that instead learn to map MLP *inputs* to MLP *outputs*. This means the transcoder learns the same function as the MLP layer but in a sparse, interpretable basis.

The key architecture is the **skip transcoder**:
```
f(x) = W_dec @ TopK(W_enc @ x + b_enc) + W_skip @ x + b_skip
```
Where:
- `x` is the MLP input (post-layernorm activation)
- `W_enc` maps to a high-dimensional dictionary (overcomplete)
- `TopK` keeps only the top-k activations, zeroing the rest (enforces sparsity)
- `W_dec` maps sparse features back to the MLP output space
- `W_skip @ x + b_skip` is an affine skip connection that handles the "easy" linear part, so sparse features only need to capture nonlinear residuals
- `W_dec` and `W_skip` should be zero-initialized; `b_skip` should be initialized to the empirical mean of MLP outputs

The training loss is simply MSE between the transcoder output and the true MLP output. No L1 penalty is needed because TopK enforces sparsity directly.

**This technique has only been applied to language models.** Your task is to apply it to a vision transformer and evaluate whether transcoders produce more interpretable features than a baseline SAE.

## Your Task

1. **Implement a skip transcoder** following the architecture described above. It should:
   - Take MLP inputs (768-dim) and predict MLP outputs (768-dim)
   - Use an overcomplete dictionary (at least 4096 features, matching the baseline SAE)
   - Use TopK sparsity with k=32 (matching the baseline)
   - Include the affine skip connection with proper initialization

2. **Train the transcoder** on the provided MLP input/output pairs. Monitor reconstruction MSE on the validation set. Your transcoder's validation MSE should be competitive with or better than the baseline SAE's reconstruction MSE.

3. **Evaluate interpretability** by measuring how well individual transcoder features align with known visual concepts:
   - For each transcoder feature, compute its activation on all validation examples
   - For each concept in concept_labels_val.csv, compute the AUROC of using each feature's activation as a predictor of that concept
   - Find the best feature for each concept (highest AUROC)
   - Compare these AUROCs against the baseline SAE's best feature per concept

- Monitor both MSE and L0 sparsity (number of nonzero features per input) during training.
